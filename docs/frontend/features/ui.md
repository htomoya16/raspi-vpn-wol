# UI Feature

## 目的

- PC管理ダッシュボードの画面構成と、レスポンシブ時の表示方針を明確化する。

## 変更内容

- `App` を入口に、表示幅で `DesktopWorkspace` / `MobileWorkspace` を出し分ける構成にした。
- PC表示は左右2カラム:
  - 左: `PC一覧` / `PC登録` タブ
  - 右: `ジョブ` / `ログ`
- モバイル表示は下部ナビ:
  - `PC一覧` / `ログ` / `PC登録` / `稼働時間` / `設定` を表示。
  - `設定` は下部ナビ右端アイコンから、他タブと同様に1画面として表示する。
- PC版はヘッダー右側の設定アイコンから設定オーバーレイを開ける。
- 設定画面は `左メニュー + 右コンテンツ` 構成:
  - 左メニュー: `テーマ色` / `外観`
  - 右コンテンツ: 選択中メニューに応じた設定項目を表示
- 設定画面でテーマ色（`デフォルト` + 10色）を切替可能にし、外観モード（`システム` / `ダーク` / `ライト`）も選択できる。
- `デフォルト` テーマは白黒基調で、外観が `システム` の場合は端末設定に応じて `ダーク（黒基調）/ ライト（白基調）` を切り替える。
- テーマはボタン色だけでなく、背景グラデーション・パネル境界・フォーカス色などのUI変数にも反映する。
- `PC一覧` / `操作ログ` / `オンライン集計グラフ` / `稼働タイムライン` / `ジョブ状態` は、共通カラートークン（surface / table / chart / timeline / state badge）経由で配色する。
- 外観モード切替時は、上記トークンを一括で切り替える設計にし、文字色と背景のコントラストを保つ。
- `クリア` / `再読み込み` / `ステータス確認` / 稼働時間の矢印ボタン（`<>`）は共通ボタンスタイル（`btn--soft` / `btn--danger`）でテーマ連動する。
- 通知（`feedback--notice` / `feedback--error`）もテーマ・外観に合わせて可読性重視で色を切り替える。
- 選択内容は `localStorage`（`wol:theme-color` / `wol:appearance-mode`）に保存する。
- PC表示では `ダッシュボード` と `稼働時間` を左右スライドページで切替可能（固定矢印ボタン）。
- `PcList` は行タップで詳細ダイアログを開く構成。
- `PC一覧` ヘッダにステータス凡例を表示:
  - `online`: 到達可能
  - `offline`: 応答なし
  - `unknown`: 未確認
  - `booting`: 起動中
  - `unreachable`: 到達不可
- 削除・ログ消去は確認ダイアログを挟み、誤操作を防ぐ。
- `操作ログ` は `前面表示` ボタンでオーバーレイ拡大表示でき、閲覧時の可読性を高める。
- `操作ログ` の `定期ステータス確認` グループは初期状態で折りたたみ表示とし、通常ログの可読性を優先する。
- フロント構造の保守性向上のため、責務分割を実施:
  - `LogsPanel` は `log-panel/` 配下へ分割（表示本体 / グルーピング / 状態管理 / sticky制御）。
  - `PcList` はヘッダ・フィルタ・一覧描画を `pc-list/` 配下へ分割し、ダイアログ状態管理をフック化。
  - `usePcData` は `usePcFilters` / `usePcCollection` / `usePcCrud` に分割。
  - `useUptimePanelState` は取得責務を `useUptimeSummaryData` / `useUptimeTimelineData` に分離。
  - `logs-panel.css` は `base / desktop / mobile` の3層に分割。
  - `App` から Workspace への受け渡しは `dashboard` オブジェクト（`workspace/types.ts`）に集約し、propsドリルを抑制。
  - テーマ管理は `theme/` 配下に分割し、`App` は `applyAppTheme()` 呼び出しに限定（詳細は `theme-system.md`）。
  - モバイル下部ナビは `MobileWorkspace` の定義配列（`MOBILE_NAV_ITEMS`）から描画する方式にし、順序変更/追加を容易化。
  - 設定メニュー（`テーマ色` / `外観`）は `SETTINGS_SECTIONS` 定義から描画し、表示切替ロジックを単純化。
- 稼働時間パネルのUI仕様:
  - `オンライン集計グラフ` は `日次 / 月次 / 年次` 切替。
  - `稼働タイムライン` は 1時間刻み縦軸・拡張高さで可読性を優先。
  - UI実装は `UptimeSummarySection` / `UptimeTimelineSection` / `UptimeToolbar` に分割し、表示責務を分離。
  - モバイルでは `稼働タイムライン` を1日ずつ表示し、左右スワイプで日次移動（週またぎ移動を含む）。
  - モバイルでは矢印ボタンを表示せず、スワイプ操作を主操作とする。
  - 初回ロードはオーバーレイ＋ `・・・`、背景更新は200ms遅延の回転スピナー表示にしてチラつきを抑制。

## 運用時の注意点

- 新しいUI要素を追加する場合は、PC版とモバイル版の両方で表示/操作を確認する。
- ダイアログ系はスクロール干渉と下部ナビ重なりを優先確認する。
- スワイプ操作を変更する場合は、縦スクロールとの干渉（誤検知）を実機幅で確認する。
